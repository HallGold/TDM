流程环节脚本说明

1．   脚本引擎内置变量

var  flowContext={};//流程上下文对象

var  bizData={};//业务对象

var  returnCode=1; //0 失败，有提示    1成功继续     2成功但不继续执行
var  resultMsg=””;//不继续运行时消息提醒
 
2.脚本引擎内置方法

HTTP.get(“url”);//发送一个get网络请求  入参：url地址

HTTP.post (“url”,object);//发送一个post网络请求入参：url地址,post数据对象

HTTP.getInside(“url”);//发送一个get内部网络服务器请求

HTTP.postInside(“url”,jsonStr);//发送一个post内部网络服务器请求,jsonStr参数为json字符串

System.out.print(“-----”);//输出打印控制台

ProcessUtils. hasClaim(“actInstId”，“nodeId”);//判定当前流程环节任务是否被领取 返回 true 或false

ProcessUtils.startFlow(“flowcontent”);// 发起一个流程  
注：flowcontent为json格式的字符串   flowcontent对象的属性请参考后面

 

3.脚本引擎示例

示例1：改变业务对象的值，并发送一个网络请求（业务对象里的a赋值‘tianding’，发送post请求并传递了三个参数 分别是流程上下文，业务对象和回调地址）

bizData.a="tianding";HTTP.post("http://localhost:8071/bpm/common/callback",{"flowContext":flowContext,"bizData":bizData,"callClass":"cn.icepanda.biz.software.flow.BizDefaultService"})

 

示例2：发送一个HTTP的get请求根据返回结果判定当前流程是否继续执行,不继续执行时返回“掉接口报错啦”

var result=HTTP.get("http://localhost:8071/bpm/definition/get?id=9ac39438613742ffb0eea252fb924b0c"); if(result.code==1){ returnCode=1}else{ returnCode =0;resultMsg="掉接口报错啦";}

 

示例3：调流程引擎内置方法，查询当前环节任务是否被领取并打印输出结果

var hasClaim=ProcessUtils.hasClaim(flowContext.actInstId,flowContext.nodeId);System.out.print("-------返回结果-------"+hasClaim);
 

示例4：脚本里发起子流程 ，并打印结果，注意如果发起的是同步子流程，请returnCode设置为2

var context=ProcessUtils.startFlow
(JSON.stringify({"actDefKey":"TEST_TIANDING_07","flowComment":"strin","flowMessage":"string","operateType":"submit","operater":"tianding"}));System.out.print("-------返回结果------"+JSON.stringify(context));returnCode=2;
 
示例5：发送一个内部服务请求(内部服务指通过网关调用，本微服务平台内的url请求，地址规范：http://服务名/服务地址，参考如下)
var result=HTTP.getInside("http://biz/biz/BlacklistConfig/timing?type=SZZSHMD"); if(result.code==1){ returnCode=1}else{ returnCode =0;resultMsg="掉接口报错啦";}

发起流程/子流程可传递参数对象

bizFlowContext{

@ApiModelProperty(value = "流程定义key")
private String actDefKey;
@ApiModelProperty(value = "操作人code")
private String operater;
@ApiModelProperty(value = "操作类型")
private String operateType = "submit";
@ApiModelProperty(value = "分支变量")
private String processVar;
@ApiModelProperty(value = "流程批示意见")
private String flowComment;
@ApiModelProperty(value = "流程消息")
private String flowMessage;
@ApiModelProperty(value = "环节ID")
private String nodeId;
@ApiModelProperty(value = "初始流程传递变量 关联业务，非必填")
private String callActTransmitVar;
@ApiModelProperty(value = "业务回调class，非必填")
private String bizCallClass;
@ApiModelProperty(value = "业务数据，非必填")
private String bizData;
@ApiModelProperty(value = "父流程上下文，为空说明非子流程")
private String parentFlowContext;
@ApiModelProperty(value = "父流程业务数据ID，为空说明非子流程")
private String parentBizDataId;
@ApiModelProperty(value = "子流程执行类型(同步异步类型sync;异步async)")
private String childrenProcessType;

}
